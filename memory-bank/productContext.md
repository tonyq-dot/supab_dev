# Product Context

Продуктовый контекст: пользователи, сценарии, приоритеты.

---

## Пользователи

### Аниматор
- Вносит работы (проект, вид работы, количество, дата)
- Прикрепляет превью к работе
- Видит свои баллы и прогноз премии
- Загружает в галерею, добавляет версии
- Выбирает тип работы для баллов (2D Scene, 3D Scene и т.д.)

### Менеджер
- Управляет проектами, видами работ
- Редактирует любые работы
- Настраивает периоды и нормы
- Экспортирует отчёты в CSV/Excel
- Управляет API ключами

---

## Потоки

1. **Добавить работу** — AddWork: проект → вид работы → количество → дата → превью (опционально)
2. **Галерея** — загрузка медиа, привязка к работе, выбор типа для баллов
3. **Отчёты** — Детальный отчёт по кварталам, экспорт

---

## Приоритеты

- Надёжность: обработка ошибок, feedback
- Удобство: понятные названия, «Привязать к работе» с информативным списком
- Безопасность: RLS, API key validation
